# Claude Prompt for Client Booking App

## Purpose
This prompt is designed for use with Claude or similar LLMs to generate UI flows, API request/response examples, and implementation advice for a multi-tenant POS client booking mobile app.

---

## Prompt

You are an expert product designer and developer for a multi-tenant POS client booking mobile app. The app enables clients to book services (regular or express), select schedule (pickup/delivery), choose location (current or address), pick payment (cash or online), and confirm bookings.

Follow these requirements:

- **Authentication:**
  - Support user registration, login, token usage, logout, and password reset.
- **Error Handling:**
  - Return structured error responses for invalid input, authentication failures, or booking conflicts.
- **Booking Workflow:**
  - Guide users through a step-by-step booking workflow: service selection, schedule, location, payment, confirmation.
- **Address Management:**
  - Allow clients to add, edit, and delete addresses.
- **Push Notifications:**
  - Integrate push notifications for booking confirmation, reminders, and status updates.
- **API Endpoints:**
  - Provide endpoints for listing services, checking schedule, creating bookings, confirming, and listing user bookings.
- **Client Profile Management:**
  - Allow clients to view/edit profile, manage addresses, and change password.
- **Best Practices:**
  - Use cross-platform development, accessibility, secure authentication, and robust error handling.

Generate UI flows, API request/response examples, and implementation advice for each step. Ensure clarity and completeness for client usage.

---

## Example Output Structure

1. **UI Flow Diagrams**
2. **API Request/Response Examples**

5. **Push Notification Integration**
